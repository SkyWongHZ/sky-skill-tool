# 周报整理 Skill

## 角色
前端工程师，每周向 team leader 汇报工作进度。

## 执行步骤

### Step 1：确定时间范围
用系统时间获取当前日期，计算**上一个完整工作周（周一～周五）**的日期范围：
- 若今天是周一，则上周为 `今天 - 7天` 的周一至周五
- 输出文件名格式：`{年}.{月}.{日}-{年}.{月}.{日}周报整理.md`
  - 示例：`2026.3.9-2026.3.13周报整理.md`（月份和日期不补零）

### Step 2：读取原始文件
在**当前工作目录**查找原始周报文件：
- 优先找文件名不含"整理"二字的 `.md` 文件
- 若用户已指定文件路径，则直接使用

### Step 3：整理内容规则
- **合并同类项**：将相同业务方向的工作归入同一分类（如"设备相关"、"流量卡需求"、"定制需求"）
- **进度标注**：
  - 已完成并上生产 → 末尾加 `✅`
  - 开发中 → 注明百分比，如 `开发中 60%`
  - 已完成未上线 → 注明"已完成"
- **语言风格**：简洁清晰，不啰嗦，不重复描述

### Step 4：输出格式
```
### 周报 {开始日期} - {结束日期}

#### {类别名}
• {工作项}：{进度/状态}

#### {类别名}
• {工作项}：{进度/状态}
```

> 注意：列表项使用 Unicode 实心点 `•`（U+2022），**不使用** Markdown 的 `-` 或 `*`，避免钉钉日报渲染乱码。

### Step 5：写入文件
将整理结果写入当前目录，文件名为 Step 1 确定的名称，同时在对话中输出完整内容，方便直接复制粘贴到钉钉日志。
